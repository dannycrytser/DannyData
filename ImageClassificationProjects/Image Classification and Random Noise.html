IMAGE CLASSIFICATION WITH TENSORFLOW/KERAS¶

The purpose of this little project was to familiarize myself with the tensorflow Python library and also Keras (which is a library that makes tensorflow easier to use, in addtion to an API). Toward the end I started tinkering with adding error to the test images to see what the effect was on the classification accuracy.

I used a standard dataset for image classification, fashion-mnist. This consists of 16 x 16 images of articles of clothing (shoes, shirts, etc), labeled by the type of article.

The first part of the notebook is just a little bit of warmup to see how the dataset works, seeing how we can extract individual images from it, and so on. We scale the pixel values in order so that they all fall in the range 0 to 1.

After the data is put in usable shape, we train the classifier. This is a neural network and so it has layers of nodes connected by links which are fired according to activation ruyles.

Keras builds classifiers step by step: first you specify how many layers you want and how many nodes should be in each layer, as well as how each layer should be connected to the previous layer (ours is Densely connected -- every possible node connection is allowed). Then we compile the model: specify how it will operate, meaning what loss function it will attempt to minimize, what algorithm it will use to try to minimize this loss function, and how its final performance will be evaluated. After compiling we fit the model, obtaining something we can actually use to classify images. Our test accuracy is about 86 percent. After we have the model we use it to classify a few images, and then we start playing with error. b

Because it takes so long to run the classifier, I didn't add the error to all the images, only a small segment of the test set. A very very small error (eps = 0.01 or so) does not seem to make much difference. There is a tiny region of error (past 0.01 but less than 0.2) in which you sometimes see better classification performance. For larger errors (eps >> 0.2) the performance is worse. My hypothesis for why you sometimes see a slight bump is that many of the images might have idiosyncratic features which are "fuzzed away" by the noise, leaving a more generic-looking image that is easier to classify.

#This specifies the size of the model for us 
#(it is still "empty" as it has not been compiled or trained)
# Think of this as the blueprint for the model before it has been built
# Each Dense layer is fully connected to the previous layer
# The integer argument specifies the number of nodes
# The activation is set to Rectified Linear Unit = (0.5)*(x+|x|)
# Which is the standard activation for each node, 
# replacing the earlier standards of sigmoid and hyperbolic tangent.

model  = tf.keras.Sequential()
model.add(tf.keras.layers.Dense(30,activation=tf.nn.relu,input_shape=(784,)))
model.add(tf.keras.layers.Dense(20,activation=tf.nn.relu))
model.add(tf.keras.layers.Dense(10,activation=tf.nn.softmax))

## Now we "compile" the model -- we actually build it!

# We also specify a loss function (this is what the NN will try
# to minimize during training), an optimizer (this is the 
# method by which the NN tries to minimize the loss),
# and a metric (this is the metric by which we judge the performance
# of the algorithm at its assigned task -- in this case classification)
# The difference between loss and non-accuracy is a little subtle
# Basically the job of the loss function is to be something that is 
# easy to calculate and optimize, and hopefully by minimizing loss
# we get something that maximizes accuracy. 

# Think of this as putting in the wiring and plumbing and light fixtures

model.compile(loss = 'categorical_crossentropy',
              optimizer = 'rmsprop',
              metrics = ['accuracy'])

## Our idea is that by adding a small amount of error to an image, we can actually improve the 
## performance of an image classifier. (Maybe this is untrue)

## Remember that all of our images are stored as lists of pixel intensities. 

def add_error(image, error = 1):
    return np.array([t + 2*(error)*(random.random()-0.5) for t in image])

def add_positive_error(image,error=1):
    return np.array([t+error*random.random() for t in image])

def add_negative_error(image,error=1):
    return np.array([t-error*random.random() for t in image])

## the function adds random error (uniformly drawn from [-error,error]) to each of the pixels in the image
## then it turns this back into the appropriate data type and asks the model to predict the class of the image. 
## We repeat this no_iterations many times, then return the largest one. (We smooth a bit randomly to break ties)
## the following function has four arguments. the first is the model (which we trained above, but in principle
## could be replaced with a different model), then an np array (just a list of numbers basically),
## then an error parameter with a default value of 0.5, then a parameter setting the number of iterations


def randomize_classification(model,nparray, error=0.5,no_iterations = 10):
    counts = [0]*10
    for i in range(no_iterations):
        vector = np.array([add_error(nparray,error)])
    
        predictions = model.predict(vector)
    
        index = np.argmax(predictions)
        counts[index]+=1
    counts = [t + 0.001*random.random() for t in counts]
    return np.argmax(counts)

num = randomize_classification(model, train_features[10])

## This function basically uses the previous one many times on the first num images in the MNIST set. 
## I guess it's just better to grab a random set of indices rather than the first num many, 
## but it's easier to do this. 

## For each of the num images, we grab the feature then have the previous function add error and classify it. 
## If the error classification is the same as the actual class, then we add one to the count_correct. Otherwise
## We add one to the count_wrong. 

## Then we return the proportion of images that the function classifies as correct. 


def correct_proportion(model, df, error=0.5, num = 40):
    count_correct = 0
    count_wrong = 0
    for i in range(num):
        feature = df[i]
        feature_array = np.array(feature)
        error_class = randomize_classification(model, feature_array,error)
        true_class = np.argmax(train_labels[i])
        if error_class == true_class:
            count_correct += 1
        else:
            count_wrong += 1
    ratio_correct = count_correct/(count_correct + count_wrong)
    
    return ratio_correct

## The following is just a list of different error amounts. 

error_range = [0.01*x for x in range(20)]

## Then for each of those error parameters we run the correct proportion function and

correct_rates = []
for t in error_range:
    z = correct_proportion(model, train_features,t,10)
    correct_rates += [z]
    print("With an error of ",t," the correct proportion is ", z)
